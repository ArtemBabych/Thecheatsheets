HTML

In this section, you can learn or refresh your knowledge of Hyper Text Markup Language.

HTML - the simple language that provides you an opportunity to create a website. First of all, it was designed to create a structure and fill in the site with content like text, images, video, and audio, and connect them with hyperlinks to create a huge system of many pages of your own website. Or just chain it with the web as a whole.

At its core, HTML revolves around the concept of elements. Each HTML element represents a distinct part of your content and is surrounded by tags, denoted by angle brackets. These tags define the structure and semantics of your document, enabling browsers to interpret and display the content appropriately.

Here are some key HTML elements that form the backbone of web development:

  • Headings;
  • Paragraphs;
  • Images;
  • Links;
  • Lists;
  • Tables;

HTML's strength lies in its versatility. Whether you're designing a personal blog, an e-commerce platform, or a portfolio site, HTML provides the foundational structure for all types of content. As you continue your exploration of HTML, you'll discover its seamless integration with CSS (Cascading Style Sheets) to enhance the visual presentation of your pages. Stay tuned to delve deeper into the world of HTML, understanding its nuances and unlocking its full potential for crafting dynamic and captivating web experiences.

Elements and Structure

HTML Element

An HTML element is a fundamental building block of an HTML document, defining the structure and content of a web page. HTML (HyperText Markup Language) uses a system of tags to encapsulate and organize different types of content within a document. Each HTML element is comprised of a start tag, content, and an end tag, with the content representing the actual information to be displayed.

HTML Tag

An HTML tag is a component of HTML used to define and structure content on a web page. HTML tags are enclosed in angle brackets (< and >) and come in pairs: an opening tag and a closing tag. The opening tag denotes the beginning of an element, and the closing tag signifies the end of that element.

<html> Element

The <html> element is a main HTML element that represents the root of an HTML document. All other HTML elements are nested within it. This element contains the entire content of an HTML document, including the document's structure, metadata, and the actual visible content.

Anchor Element

The <a> element, also known as the "anchor" element, is used to create hyperlinks in HTML. Hyperlinks are references to other web pages, documents, or resources, allowing users to navigate between different pages on the internet.

Head Element

The <head> element is an integral part of an HTML document and is used to contain metadata and other head elements that are not part of the main content. It's typically placed within the <html> element and before the <body> element.

Target Attribute

The <target> attribute is used in the <a> (anchor) element in HTML to specify how the linked resource should be displayed when the user clicks on the link. The target attribute determines the browser context in which the linked content will open.

Indentation

Indentation in HTML refers to the practice of adding spaces or tabs at the beginning of lines of code to visually organize and structure the code. While indentation is not strictly necessary for the browser to interpret and render HTML, it is essential for making the code more readable and maintainable, especially as web development projects become more complex.

Link to a Different Part of the Page #

To create a link to a different part of the same page, you can use the <a> (anchor) element along with the id attribute.

Comments

In HTML, comments are used to include explanatory notes or annotations within the code that are not visible in the rendered output of the web page. Comments can be useful for documenting your code, providing context for other developers, or temporarily disabling parts of the code for testing purposes. HTML comments are written using the <!-- and --> delimiters. Anything between these delimiters is treated as a comment and is ignored by the browser when rendering the web page.

Whitespace

Whitespace in HTML refers to spaces, tabs, and line breaks that are used to format and organize the code for better human readability. In HTML, most whitespace is generally ignored by browsers when rendering the content, meaning that extra spaces and line breaks do not affect the appearance of the web page.

Title Element

The <title> element in HTML is used to define the title of the HTML document. The text within the <title> element appears in the browser's title bar or tab and provides a brief description of the content of the page. Additionally, search engines may use the title as part of the search results.

File Path

File paths are crucial in web development, serving as addresses to locate resources like HTML files, images, stylesheets, and scripts. There are two main types of file paths: absolute and relative. Absolute Paths: These paths provide the complete file location from the root directory. They include the entire file path, starting from the root of the file system. Relative Paths: Relative paths specify file locations based on the current directory. If you need to use the current folder you can write "./".

Document Type Declaration

The <!DOCTYPE html> is an important declaration in HTML that specifies the version of HTML being used in a document. It helps browsers to interpret and render the content correctly. This is placed at the very beginning of an HTML document, before the <html> tag.

Element Content

In HTML, the term "element content" refers to the content enclosed within an HTML element. HTML elements are composed of a start tag, content, and an end tag. The content is the actual information or structure that you want to include within that element.

List Item Element

The <li> (List Item) element in HTML is used to define items within ordered lists (<ol>) and unordered lists (<ul>). Each <li> element represents an individual item in the list.

Video Element

The <video> element in HTML is used to embed video content into a web page. It provides a standardized way to include video files and offers controls for users to play, pause, adjust volume, and more.

Emphasis Element

The <em> element in HTML is used to indicate emphasis on text, typically by rendering the enclosed text in italic or with some other form of emphasis based on the browser's default styling.

Div Element

The <div> element in HTML is a generic container that is used to group together and structure content on a web page. The <div> element itself does not have any specific semantic meaning; its purpose is mainly for organizing and styling content.

HTML Structure

HTML (HyperText Markup Language) provides a structured way to organize content within a web page. The basic structure of an HTML document includes essential elements that define the document type, set the language, and structure the content.

Attribute Name and Values

In HTML, attributes provide additional information about an element and are always included in the opening tag. The syntax for an HTML attribute is name="value".

Line Break Element

The line break in HTML is represented by the <br> (break) element. It is a self-closing tag that is used to insert a line break within text. The <br> element is commonly used when you want to force a new line without starting a new paragraph.

Image Element

The <img> (image) element in HTML is used to embed images into a web page. It is a self-closing tag, meaning it doesn't have a separate closing tag.

Heading Elements

HTML provides heading elements to structure content hierarchically. Headings range from <h1> (the highest level) to <h6> (the lowest level). Using heading elements appropriately helps create a clear document structure. Search engines and assistive technologies use heading tags to understand the hierarchy and importance of content on a page. It's good practice to use headings semantically, with <h1> for the main heading and subsequent levels reflecting the structure of your content.

Paragraph Element

The <p> (paragraph) element in HTML is used to define paragraphs of text. It is a block-level element that represents a separate block of text and is commonly used to structure textual content on a webpage.

Unique ID Attributes

In HTML, the id attribute is used to uniquely identify an element on a web page. The id attribute should have a unique value within the entire HTML document, meaning no two elements should share the same id value. This uniqueness is crucial for various purposes, including styling with CSS, scripting with JavaScript, and linking to specific elements. A few tips: Start with a letter (a-z, A-Z). Follow with letters, digits (0-9), underscores (_), hyphens (-), or periods (.).

Body Element

The <body> element in HTML represents the content of an HTML document. It contains the main content that is displayed in the browser, such as text, images, links, and other elements. The <body> element is a block-level element, and it is a required part of every HTML document.

Span Element

The <span> element in HTML is an inline container that is used to apply styles or scripting to a specific portion of text within a larger block of content.

Strong Element

The <strong> element in HTML is used to give strong importance to its enclosed text, indicating that the text has strong importance, seriousness, or urgency. Visually, browsers commonly render text within <strong> as bold, but the primary purpose of the element is to convey semantic meaning rather than specific styling.

Tables

In HTML, there are several elements related to tables. Here are the primary table-related elements:

  1. <table>: Defines the start of a table.
  2. <tr>: Represents a table row.
  3. <th>: Defines a header cell in a table.Typically used in the first row or first column.
  4. <td>: Defines a data cell in a table.

Additionally, there are elements that help structure the content within a table:

  1. <thead>: Groups the header content in a table.
  2. <tbody>: Groups the body content in a table.
  3. <tfoot>: Groups the footer content in a table.

Semantic HTML

Semantic HTML, or structural HTML, plays a vital role in web development by providing meaning and context to the content within a web document.

In the HTML language, semantic elements such as <header>, <nav>, <main>, <section>, <article>, <aside>, and <footer> serve as the building blocks for creating well-organized, accessible, and SEO-friendly web pages.

Similar to HTML's structural elements, semantic elements go beyond mere presentation and contribute to the overall structure and comprehension of the document.

Embracing semantic HTML enhances clarity, readability, and accessibility for both humans and machines, fostering better search engine optimization.

It is a fundamental practice in modern web development, emphasizing the importance of conveying meaningful information about the content and its structure.


Semantic tags
Element Description More Information
<header> Represents the header of a section or a page. Typically contains headings, navigation links, or introductory content. Enhances accessibility and provides a structural container for introductory content.
<nav> Represents a navigation menu or links. Typically used for site navigation. Helps define the navigation section of a page, making it easier for screen readers to identify.
<main> Represents the main content of the document. Typically excludes headers, footers, and sidebars. Improves document structure, aiding in page navigation and screen reader interpretation.
<section> Represents a thematic grouping of content. Often used to group related content together. Provides a semantic container for organizing content into distinct sections.
<article> Represents a self-contained piece of content that can be distributed and reused. Examples include blog posts, news articles, or forum posts. Helps search engines identify and index independent content units, improving search results.
<aside> Represents content that is tangentially related to the content around it. Often used for sidebars or content like related links. Provides a designated space for content related to the main content but not central to it.
<footer> Represents the footer of a section or a page. Typically contains metadata, copyright information, or links to related documents. Conveys important information about the document, improving accessibility and user experience.
<figure> Used to encapsulate media content, such as images or videos. Facilitates the association of images or multimedia with explanatory content.
<figcaption> Used to provide a caption for the content within the <figure>. Improves accessibility by associating descriptive text with images or multimedia content.
